# 隐私与网络边界

本文描述当前本地优先实现的数据落盘、出站请求、凭据和备份边界。它是实现说明，不替代所选招聘平台或模型服务商的隐私条款。

## 本机保存的数据

Fuck Job 使用 Tauri 的平台目录 API，不在代码或文档中假定固定的 OS 路径：

- `app_config_dir/app_config.yaml`：招聘筛选、话术、浏览器路径、LLM 地址/模型/超时和可选简历上下文等配置。
- `app_data_dir/data/`：`job_details.json`、`chat_messages.json`、`interview_analyses.json` 和 `user_resumes.json`。
- `app_data_dir/browser-profile/`：默认的独立 Chromium 资料目录，可能含招聘平台登录状态和网站存储。
- `app_data_dir/rpa.log`：运行状态与错误日志。
- `app_data_dir/backups/`：迁移保护副本；`app_data_dir/recovery/`：每次恢复前创建的恢复点。

“关于与数据”会显示当前系统解析出的应用数据目录。配置和 JSON 数据采用临时文件、同步和原子替换方式写入；备份/恢复期间使用进程内排他锁避免与普通写入交错。

## 发送给所配置 LLM 的数据

只有在用户主动获取模型列表、测试连接或运行 AI 功能时，应用才访问配置的 OpenAI 兼容地址：

- 模型列表：向 `<base_url>/models` 发起请求。
- 短连接测试：向 `<base_url>/chat/completions` 发送固定的简短测试提示。
- 流式测试：向同一路径发送固定测试提示并读取 SSE。
- 打招呼/岗位分析：可能发送岗位标题、公司、薪资、地点、岗位描述和配置的提示词。
- 回复生成/沟通调试：可能发送聊天历史、岗位描述、背景上下文和提示词。
- 简历相关 AI：会发送用户选择或编辑的简历内容、问题、回答或模拟面试历史。
- 开启“注入简历上下文”后，自动打招呼或回复也可能携带配置中的简历内容。

请求使用配置的模型 ID；若存在有效密钥，会使用 Bearer Authorization。应用无法控制远程服务商如何保存、训练或转发这些内容。对敏感数据，应使用本机 Ollama/LM Studio，或选择有合适数据政策的服务商并最小化上下文。

## 招聘平台流量

仅在用户执行环境检查、扫码登录、抓取已沟通过岗位、单轮/周期求职或回复未读时，自动化浏览器访问以下站点：

- BOSS 直聘：`zhipin.com` / `www.zhipin.com`。
- 猎聘：`liepin.com` / `www.liepin.com`。

这些请求由受控 Chromium 会话发出，行为和普通浏览器访问一样会受平台 Cookie、风控、账号状态及服务条款约束。应用不把招聘数据转发到原项目业务服务器。

## 用户主动打开的链接

“关于与数据”中的仓库、Apache-2.0 许可和文档按钮会调用系统默认浏览器。用户点击简历或其他内容中的外部链接时，也可能由系统浏览器访问目标站点。这些导航由用户触发，不是后台遥测。

## 密钥存储与优先级

模型密钥的解析顺序为：

1. 系统钥匙串/凭据库中的 `fuck_job` / `llm_api_key`。
2. 环境变量 `FUCKJOB_LLM_API_KEY`。
3. 无密钥。

若读取系统凭据库失败，但 `FUCKJOB_LLM_API_KEY` 是非空值，应用仍会使用环境变量并将凭据来源报告为“环境变量”。若凭据库读取失败且环境变量缺失或为空，应用会提示可配置该环境变量。

界面“保存凭据”写入系统凭据库；“清除模型密钥”只删除该凭据库条目，不修改环境变量。Tauri 命令只返回“是否配置”和来源，不返回明文密钥。Ollama、LM Studio 默认不要求密钥；自定义端点是否需要密钥取决于服务端。

## 日志与脱敏

`app_data_dir/rpa.log` 记录时间、级别、平台和操作状态。写入前会对 Authorization、API Key、token、Cookie、简历正文、聊天正文和 prompt 等常见键值形式做脱敏。错误对 UI 只返回分类码和稳定的用户消息；Rust 内部诊断详情不会序列化到 Tauri 命令结果。模型服务的非成功 HTTP 响应及 SSE/JSON 解析错误仅在内部保留 HTTP 状态、经过字符和长度约束的服务商错误 code/type，以及解析类别和位置，不保留任意响应正文。

脱敏是纵深防御，不应视为任意文本的完美数据防泄漏。第三方页面或库错误可能包含意外格式；分享日志前仍应人工检查。可以在“关于与数据”中清空日志。

## 备份内容与排除项

导出 ZIP 的允许列表只有：应用配置、岗位、聊天、分析和用户简历数据，以及包含版本与 SHA-256 的清单。导出时会递归移除配置中名为 API Key、Authorization、Cookie、token 等敏感字段。

以下内容明确不进入导出备份：

- 系统钥匙串/凭据库和 `FUCKJOB_LLM_API_KEY`。
- `rpa.log`。
- `browser-profile/` 及招聘平台登录态/Cookie。
- 迁移备份、恢复点和允许列表外的任意文件。

恢复会拒绝路径穿越、符号链接、重复/未知文件、缺失文件、校验和错误、数据文件类型/字段结构错误和未来版本。旧版配置会先在内存中迁移、验证并重新序列化为不含密钥的当前格式。所有内容验证通过后才开始写入；先生成恢复点，写入失败时回滚所有目标文件。恢复成功后需手动重启。

## 明确不存在的后台连接

当前运行时不包含：

- 原项目业务服务器、旧账号/登录/注册、积分或兑换 API。
- 遥测、崩溃上报或云端同步。
- 自动更新检查、更新清单或 Tauri updater。
- DashScope/其他提供商的联网搜索开关，或任何 Web 搜索功能。

## 自行审计

在仓库根目录运行：

```bash
pnpm check:network
```

该脚本扫描前端、Rust 运行时、Tauri/Cargo 配置、`package.json` 和 `build.sh`，并在发现原服务器、旧接口、硬编码搜索或更新器引用时失败。测试文件、文档、依赖锁和构建产物不会被当成运行时。

可补充查看所有 URL 和敏感日志调用：

```bash
rg -n --glob '!node_modules/**' --glob '!src-tauri/target/**' 'https?://' src src-tauri/src
rg -n --glob '!src-tauri/target/**' 'logger::|Authorization|api[_-]?key|token|cookie' src-tauri/src
```
