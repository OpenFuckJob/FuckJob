核心功能

求职全流程，一站自动化

从发现机会到拿下 offer，每个环节都有 AI 加持。

一键自动化投递

检查浏览器和登录状态后，自动执行岗位搜索、筛选和投递。支持单次投递、回复未读、周期投递三种模式，让系统替你跑完繁琐流程。

智能岗位筛选

按关键词、薪资、经验、学历、行业、城市等多维度精准定位目标岗位。支持正则表达式高级规则，排除不相关岗位，投递效率翻倍。

AI 智能回复

基于大语言模型根据对话上下文自动生成回复内容，或使用正则模板匹配固定场景。支持文本、图片、LLM 生成三种回复方式，沟通不中断。

简历优化工作台

PDF 简历一键解析，AI 全文润色和逐章节优化。内置多个专业模板和主题配色，支持 Markdown 编辑和 PDF 导出，让简历脱颖而出。

面试深度分析

基于 JD 和你的简历自动生成匹配度分析、风险点识别、项目追问清单和面试准备建议，让你有备而来。

隐私安全保障

简历、配置、聊天记录等所有敏感数据本地加密存储。服务端仅做模型推理，不保存任何个人招聘隐私信息。你的数据，你做主。

隐私设计

你的数据，永远在你手里

这不是一句口号，而是架构级别的设计选择。

本地优先架构

简历内容、岗位筛选规则、回复模板、运行日志、聊天上下文——全部默认保存在你的设备上。

服务端只在需要 AI 能力时接收请求做模型推理，推理完成后不存储、不缓存、不沉淀你的任何个人信息。我们把「不存储」写进了产品架构，而不是隐私条款。

本地加密存储

配置文件、简历内容和运行日志采用 AES 加密存储在本机，密钥由操作系统级密钥链管理。

推理即焚

服务端仅承担模型推理，请求处理完毕后不保留任何简历、聊天、岗位沟通上下文数据。

可控可导出

所有配置支持 YAML/JSON 导入导出，你的求职策略和数据随时可迁移、可备份。

常见问题

你可能想知道的

关于数据安全、使用方式和效果边界的回答。

服务端会保存我的简历或聊天记录吗？

不会。这是产品的核心架构原则：服务端只做模型推理，请求处理完毕后不保留任何个人数据。你的简历和聊天记录只在你本地设备上。

需要自己有 LLM API Key 吗？

需要。客户端集成了 LLM 调用能力，你需要配置自己的 API Key（兼容 OpenAI 格式），用于自动回复、简历优化和面试分析等 AI 功能。

支持哪些招聘平台？

当前聚焦 BOSS 直聘。通过浏览器自动化方式运行，不依赖平台 API，但也意味着平台 UI 变动可能需要适配更新。

会不会被封号？

合理使用风险较低。建议设置合理的投递频率，使用周期投递模式分散请求压力。工具只是辅助，自行评估使用策略和风险。

能保证拿到 offer 吗？

不能，也不应该有工具做这种承诺。Fuck Job 解决的是重复操作和信息整理的成本，让你把精力放在真正需要人的判断力的环节上。

怎么更新到最新版本？

客户端内置自动更新机制，启动时会检测新版本并提示升级。也可在交流群获取最新安装包手动更新。
